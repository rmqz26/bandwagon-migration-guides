## 搬瓦工换机房是什么意思

搬瓦工（BandwagonHost）是一家做了十几年的美国 VPS 服务商，旗下自研的 KiwiVM 控制面板提供了一个叫「Migrate to another DC」的功能，让用户在购买 VPS 之后可以把服务器从一个数据中心迁移到另一个——这就是通常说的「换机房」。

操作完成后，你的 VPS 上的系统文件、配置、服务全部原封不动搬过去，只有 IP 地址会跟着机房位置变化。

## 第二步：换机房操作全流程

支持换机房的套餐，整个操作在 KiwiVM 控制面板里完成。

**1. 登录 KiwiVM 控制面板**

进入搬瓦工官网后台，点击 Services → My Services，找到你要操作的 VPS，点击右边的「KiwiVM Control Panel」按钮。

**2. 先把 VPS 关机（部分情况下需要）**

早期版本的教程要求先 stop 掉实例再迁移，现在的 KiwiVM 更新之后，迁移过程中 VPS 可以保持运行状态，只在迁移最后阶段会有几秒到几分钟的短暂中断。保险起见，如果 VPS 上有重要服务，可以先做一个快照备份。

**3. 点击左侧菜单「Migrate to another DC」**

进入页面后可以看到当前所在机房（Current location of this VPS），下方是可选的目标机房列表。

**4. 选择目标机房，点击「Confirm on next step」**

选好机房之后点击确认，进入下一步界面。

**5. 点击「Start Migration」开始迁移**

点击之后就正式开始了。页面会显示新机房分配的 IP 地址，同时你的注册邮箱也会收到一封通知邮件，里面也有新 IP 信息。

**6. 等待迁移完成**

通常 15 分钟以内完成，数据量大的话可能稍微长一点。迁移完成后会再收到一封确认邮件。

## 换机房会有哪些影响？

这几个问题是大多数人最想知道的：

**IP 地址会变吗？**

会。换到新机房之后，搬瓦工会给你分配那个机房的新 IP。有一种情况除外：如果你从 A 机房换到 B 机房，再换回 A 机房，大概率还是原来那个 IP，除非之前的 IP 已经被其他用户占用了。

**数据会丢失吗？**

不会丢。系统、文件、配置都会随着迁移一起搬过去。迁移完成后只需要把之前用到旧 IP 的地方（比如 SSH 配置、DNS 解析）更新成新 IP 就行。

**会消耗月流量吗？**

这是容易被忽略的一点：换机房是免费的，但会消耗你的月流量，消耗量大约是 VPS 已用硬盘空间的 4 到 8 倍。如果你的 VPS 装了不少东西，流量接近告警，迁移前最好先确认一下剩余流量是否够用。

**VPS 在迁移过程中可以用吗？**

迁移大部分过程中 VPS 仍然在线，只在最后阶段会短暂断线（几秒到几分钟）。不建议在此期间做重要操作。

## IP 被封可以靠换机房解决吗？

不能，而且这是非常常见的误区。

搬瓦工的换机房功能，有一个前提条件：当前 IP 必须是正常可访问的。如果你的 IP 已经被封无法连接，KiwiVM 控制面板的迁移功能就直接用不了。

这两件事的关系是：机房迁移是用来「优化线路」的，付费换 IP 才是用来「修复被封 IP」的。被封之后想恢复，只能通过付费换 IP 的方式解决，官网的价格是每次 8.79 美元。

## FAQ：搬瓦工换机房常见问题

**Q：换机房之后 SSH 配置要重新设置吗？**

A：系统和数据都不用重新配置，只需要把 SSH 客户端里保存的旧 IP 换成新 IP 就行，端口和密码都不变。

**Q：KVM 套餐能迁移到 CN2 GIA 机房吗？**

A：不能。KVM 常规套餐只能在对应的普通机房列表里切换，CN2 GIA-E 套餐才有权限使用 CN2 GIA 机房。

**Q：换机房次数有限制吗？**

A：官方没有明确的次数限制，但建议合理使用，每次都会消耗一定量的月流量，频繁切换流量消耗不容忽视。

**Q：迁移完成后发现连不上新机房怎么办？**

A：先重启 VPS 试一下，大部分情况下重启后就能正常连接。如果还是不行，可以通过 KiwiVM 的 VNC 或 Root Shell 功能进入系统排查，或者尝试从快照恢复。

**Q：限量版套餐买完可以换机房吗？**

A：可以，但各款限量版的可迁移机房范围各不相同，购买前可以查看对应套餐的机房列表确认。
